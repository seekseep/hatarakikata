## 多様性の設計方針

以下の軸で多様性を確保している:

- **性別**: 男性12人、女性8人
- **出身地域**: 北海道〜沖縄まで全国各地（地方/都市のバランス）
- **学歴ルート**: 旧帝大、一般大学、文系大学、専門学校、高専、美大、高卒
- **キャリアパス**: 新卒エンジニア、異業種転職、独学、スクール経由、デザイナーからの転向、起業→失敗→再就職、非正規→正社員
- **専門領域**: Web、モバイル、データ、クラウド、セキュリティ、ゲーム、組み込み、マネジメント、AI/ML、QA、デザインエンジニアリング、プラットフォーム、PdM
- **ライフイベント**: 独身、既婚、離婚、子育て中、シングルマザー、事実婚、バツイチ再婚
- **文化的背景**: 在日コリアン、インドルーツなど

## ファイル構成

```
people/
├── README.md（この要件ファイル）
├── profiles/
│   ├── P01.md 〜 P20.md（個別プロフィール）
├── stories/
│   ├── P01_高橋陽太.md 〜 P20_黒田颯太.md（詳細ストーリー）
└── data/
    └── careerMaps/
        ├── 高橋陽太.json 〜 黒田颯太.json（careerMap JSON）
```

## データ形式

### profiles/（基本プロフィール）
マークダウン形式。性格・出身・家庭環境・キャリア概要などの基本情報。

### stories/（詳細ストーリー）
マークダウン形式。各時代ごとに「家族との暮らし」「学校/仕事」「人格形成に寄与した活動」「気持ち」を段落で記述。careerMap JSONの元データとなる。

### data/careerMaps/（キャリアマップJSON）
タイムライン可視化用のJSON。各イベントにtype（working/living/feeling）、期間、重要度（strength）、表示行（row）、タグを持つ。
